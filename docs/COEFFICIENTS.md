# Формула коэффициентов — LMSR (Logarithmic Market Scoring Rule)

## Зачем нужна формула

В обычных букмекерских конторах коэффициенты выставляет аналитик. На prediction market коэффициенты должны рассчитываться **автоматически** на основе того, сколько денег уже поставлено на каждый исход.

Задача: чем больше людей ставит на исход → тем дороже становится покупать акции этого исхода (коэф уменьшается, цена растёт).

## Используемая модель — LMSR

Это та же математика, что используют **Polymarket, Augur, PredictIt**. Изобретена Робином Хансоном в 2003 году.

### Базовая формула

Стоимость рынка (cost function):

```
C(q₁, q₂, ..., qₙ) = b · ln(e^(q₁/b) + e^(q₂/b) + ... + e^(qₙ/b))
```

Где:
- `qᵢ` — количество купленных акций на исход `i`
- `b` — параметр ликвидности (чем больше — тем стабильнее цены)

### Цена акции исхода (вероятность)

```
pᵢ = e^(qᵢ/b) / Σⱼ e^(qⱼ/b)
```

Сумма всех `pᵢ = 1` (т.е. это нормированные вероятности).

### Коэффициент = 1 / pᵢ

Если `pᵢ = 0.4` (40% вероятность), то коэффициент `= 2.5`.
Поставил 100₽ → выиграл 250₽ (если исход случился).

### Стоимость покупки `Δq` акций

```
cost = C(q' + Δq) - C(q)
```

То есть пользователь платит **разницу** между новой и старой стоимостью рынка.

## Пример на бинарном рынке (YES/NO)

Параметры: `b = 1000`, начальные `qYES = 0`, `qNO = 0`.

**Старт:** обе акции стоят 0.5 (50/50).

**Алиса покупает 100 YES:**
- `qYES = 100, qNO = 0`
- Стоимость: `1000 · ln(e^0.1 + e^0) − 1000 · ln(2) ≈ 51.8`
- То есть Алиса заплатила ~52₽ за 100 акций (средняя цена 0.52)
- Новая цена YES: `e^0.1 / (e^0.1 + 1) ≈ 0.525`
- Коэф YES: `1/0.525 ≈ 1.90`
- Коэф NO: `1/0.475 ≈ 2.10`

**Боб покупает 500 YES:**
- `qYES = 600, qNO = 0`
- Цена YES взлетает до ~0.646
- Коэф YES: `~1.55` (стало дороже ставить на YES)

## Параметр `b` — ликвидность

| Значение | Что означает |
|---|---|
| `b = 100` | Маленький рынок, цены прыгают резко |
| `b = 1000` | Средний (рекомендую для старта) |
| `b = 10000` | Большой рынок, цены стабильные |

**Платформа изначально дотирует рынок** — фиксированный убыток платформы по LMSR ограничен:

```
max_loss = b · ln(N)
```

Где `N` — число исходов. Для бинарного: `b · ln(2) ≈ 0.693 · b`.

При `b = 1000` максимум платформа может потерять 693₽ на одном рынке. Зато активность пользователей быстро превысит этот убыток за счёт комиссии.

## Расчёт выплат при разрешении события

Когда событие разрешается (например, "YES" победил):
- Все, у кого есть YES-акции, получают по 1₽ за акцию
- NO-акции сгорают
- Платформа берёт **комиссию** (`PLATFORM_FEE_PERCENT`) с прибыли пользователя

## Альтернативы и почему мы выбрали LMSR

| Подход | Плюсы | Минусы |
|---|---|---|
| **LMSR** ✅ | Всегда есть ликвидность, простая формула | Платформа держит риск |
| Order book (как биржа) | Нет риска платформы | Без ликвидности рынок мёртвый |
| Constant Product (Uniswap) | Используют DEX | Хуже для бинарных рынков |
| Фикс. коэффициенты | Просто | Нужны аналитики |

**Polymarket использует CPMM** (упрощённую версию AMM). Для MVP в Telegram **LMSR** удобнее, потому что не требует initial liquidity provider — платформа сама ставит `b` и стартует.
